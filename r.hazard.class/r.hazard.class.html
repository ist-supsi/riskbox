NAME

r.hazard.class - Hazard classification of area maps

KEYWORDS

hazard, classification

SYNOPSIS

r.hazard.class
r.hazard.class help
r.hazard.class [-c] [-s] low_int=string med_int=string high_int=string low_ub=value med_ub=value l_class=string m_class=string h_class=string t_class=string [--overwrite] [--verbose] [--quiet]

Flags:

-c
Set color maps
-s
Set cautelative mode
--overwrite
Allow output files to overwrite existing files
--verbose
Verbose module output
--quiet
Quiet module output

Parameters:

low_int=string
Intensity map name (low return period)
med_int=string
Intensity map name (medium return period)
high_int=string
Intensity map name (high return period)
low_ub=value
Lower intensity class upper boundary value
med_ub=value
Medium intensity class upper boundary value
l_class=string
Low return period hazard map name
m_class=string
Medium return period hazard map name
h_class=string
High return period hazard map name
t_class=string
Total hazard map classification name

DESCRIPTION

r.hazard.class allows to assess hazard maps according to the approach developed in Switzerland for natural hazards. (BUWAL 1999, OFAT, OFEE, OFEFP 1997, Raetzo et al. 2002). Two main aspects describe the level of hazard: intensity and return period. The intensity represents the magnitude of an event and is divided into three classes: high, medium and low. The return period, or probability, describe how often a hazard occurs and is also divided into three classes: low, medium and high.

Input intensity maps (one for each probability class) are classified in hazard levels according to the intensity-probability codes represented in the matrix below:


Hazard classification matrix

As a default, if a cell contains two codes (e.g. medium intensity and low return period) the lower code is selected. If the user wants to apply a conservative approach and to select the higher codes it's possible to activate the flag -s.

Encoded maps are so integrated to obtain the final hazard map, where the increasing level of hazard is identified by the following codes:

0 = no hazard
1 = low hazard
2 = medium hazard

If the flag -c is activated the hazard map is visualized with the colors represented in the matrix (no hazard areas are white).
